# KandaLedger — ค่าน้ำค่าไฟ & สั่งซื้อ แคมป์คนงาน

เว็บเดียว 2 โมดูล + ฐานข้อมูล Google Sheet + โฮสต์ฟรีบน GitHub Pages

- **โมดูลค่าน้ำค่าไฟ** — แยกตามคนงาน, มิเตอร์ไฟ/น้ำ → คำนวณหน่วย × เรต, ติ๊กจ่ายแล้ว, เลือกช่วงเดือน, พิมพ์
- **โมดูลสั่งซื้อ** — แบ่งเป็นรอบ, สรุปยอดต่อรอบ, ปุ่มพิมพ์ต่อรอบ
- **2 โหมด** — หน้าดูสาธารณะ (ใครก็ดูได้) / หน้ากรอกข้อมูล (ล็อกด้วย PIN)

```
KandaLedger/
├─ index.html        หน้าเว็บ (โครง)
├─ styles.css        ดีไซน์ระบบ (Noto Sans Thai, light/dark)
├─ app.js            ตรรกะทั้งหมด + การพิมพ์
├─ config.js         << วาง URL ของ Apps Script ตรงนี้
├─ apps-script/
│  └─ Code.gs        โค้ดฝั่ง Google (backend + ย้ายข้อมูลเดิม)
└─ README.md
```

## ✅ สถานะ: ตั้งค่าให้แล้วเกือบครบ (ผ่าน clasp)

ผมทำให้แล้ว: สร้าง Apps Script ผูกกับ Sheet → push โค้ด → deploy web app → ใส่ URL ใน `config.js` แล้ว
**เหลือขั้นเดียวที่ Google บังคับให้เจ้าของทำเอง** = กด Run `setup` 1 ครั้งเพื่ออนุญาตสิทธิ์ + ย้ายข้อมูล:

1. เปิด Apps Script editor ของโปรเจกต์คุณ (จาก Google Sheet → Extensions ▸ Apps Script)
2. แถบบนเลือกฟังก์ชัน **`setup`** → กด **Run**
3. ครั้งแรกขอสิทธิ์ → เลือกบัญชีคุณ → **Advanced** → **Go to … (unsafe)** → **Allow**
4. เสร็จแล้วจะเห็น 4 แท็บ + ข้อมูลสั่งซื้อเก่าถูกย้ายเข้า (20 รอบ 95 รายการ)
5. กลับมาเปิด `index.html` → ใส่รหัสเข้าเว็บที่ตั้งไว้ใน **Script Properties** (คีย์ `ENTRY_PIN`) · แก้สั่งซื้อใช้ `PURCHASE_PIN`

> 🔑 รหัสผ่านเก็บใน **Script Properties** (Project Settings ▸ Script Properties) คีย์ `ENTRY_PIN` / `PURCHASE_PIN` — **ไม่อยู่ในซอร์สโค้ด** (กันรหัสรั่วเมื่อ repo เป็น public). เปลี่ยนรหัส = แก้ค่าใน Properties ได้เลย ไม่ต้อง push ใหม่. URL เว็บแอปเป็น public เสมอ → **ตั้งรหัสยาว 8-10 ตัวผสมตัวอักษร+เลข**
> ถ้าแก้ `Code.gs` ใหม่: `cd apps-script && clasp push -f && clasp create-deployment -i AKfycbz3uwRgw4kSecNvzRiZ2BmHkFMu_ZfntgcFE0r8NqHao13SFcZTom1fT493h202ljeo` (redeploy ทับ URL เดิม)

ด้านล่างคือคู่มือฉบับเต็ม (เผื่อต้องทำเองใหม่ทั้งหมด)

## ขั้นที่ 1 — ตั้งค่า Google Sheet + Apps Script (ทำครั้งเดียว)

สร้าง Google Sheet เปล่าใน Google Drive ของคุณ (ชื่ออะไรก็ได้) สำหรับเป็นฐานข้อมูล

1. เปิด Sheet นั้น → เมนู **Extensions ▸ Apps Script**
2. ลบโค้ดตัวอย่างทั้งหมด แล้วก๊อปปี้เนื้อหาไฟล์ `apps-script/Code.gs` ไปวางแทน → กด **Save** (ไอคอน 💾)
3. ตั้งรหัสผ่านที่ **Project Settings (ไอคอนเกียร์) ▸ Script Properties** (ไม่ได้เก็บในโค้ดแล้ว):
   - `ENTRY_PIN` = รหัสเข้าเว็บ (ดู/แก้ ค่าน้ำค่าไฟ + ดู สั่งซื้อ)
   - `PURCHASE_PIN` = รหัสแก้สั่งซื้อ
   - แนะนำรหัสยาว 8-10 ตัวผสมตัวอักษร+เลข (URL เป็น public เสมอ PIN คือด่านเดียว)
4. ด้านบน เลือกฟังก์ชัน **`setup`** จาก dropdown → กด **Run**
   - ครั้งแรกจะขอสิทธิ์ → Review permissions → เลือกบัญชีคุณ → Advanced → “Go to … (unsafe)” → Allow
   - เสร็จแล้วจะเห็น 4 แท็บใน Sheet (`Workers`, `Bills`, `Rounds`, `Items`) และข้อมูลสั่งซื้อเก่าถูกย้ายเข้า `Rounds` + `Items` เรียบร้อย

> หมายเหตุ: คอลัมน์ “รวม” เดิมใน Sheet กรอกไม่สม่ำเสมอ (บางแถวเป็นยอดสะสม) ระบบจึง **คำนวณยอดรวมใหม่ทุกแถว** เป็น `ราคา + ค่าส่ง − ส่วนลด` ให้ตรงกันทั้งหมด แก้ไขทีหลังได้ในหน้ากรอกข้อมูล

## ขั้นที่ 2 — Deploy เป็น Web App

1. ใน Apps Script กด **Deploy ▸ New deployment**
2. ไอคอนเฟือง ⚙️ ข้าง “Select type” → เลือก **Web app**
3. ตั้งค่า:
   - **Execute as:** `Me`
   - **Who has access:** `Anyone`  ← สำคัญ (ให้เว็บเรียกได้)
4. กด **Deploy** → คัดลอก **Web app URL** (ลงท้าย `/exec`)

> ทุกครั้งที่แก้ `Code.gs` แล้วอยากให้มีผล ให้ **Deploy ▸ Manage deployments ▸** (ดินสอ) **▸ Version: New version ▸ Deploy** (URL เดิมไม่เปลี่ยน)

## ขั้นที่ 3 — ใส่ URL ลงเว็บ

เปิด `config.js` แล้ววาง URL:

```js
window.KANDA_CONFIG = {
  WEB_APP_URL: "https://script.google.com/macros/s/AKfyc..../exec",
  ORG_NAME: "แคมป์คนงาน"
};
```

เปิด `index.html` ในเบราว์เซอร์เพื่อทดสอบ — แบนเนอร์ส้ม “โหมดตัวอย่าง” ต้องหายไป และข้อมูลจริงต้องขึ้น

## ขั้นที่ 4 — ขึ้น GitHub Pages (ฟรี)

1. สร้าง repo ใหม่บน GitHub (เช่น `kandaledger`) แล้วอัปโหลด **ไฟล์ในโฟลเดอร์นี้** (ไม่ต้องเอา `apps-script/` ขึ้นก็ได้ แต่เอาขึ้นไว้เป็น backup ก็ดี)
   ```bash
   cd KandaLedger
   git init
   git add .
   git commit -m "KandaLedger web app"
   git branch -M main
   git remote add origin https://github.com/<user>/kandaledger.git
   git push -u origin main
   ```
2. ไปที่ repo → **Settings ▸ Pages**
3. **Source:** Deploy from a branch → **Branch:** `main` / `/ (root)` → Save
4. รอ ~1 นาที จะได้ลิงก์ `https://<user>.github.io/kandaledger/`

เปิดลิงก์ครั้งแรกจะ **ถามรหัสเข้าเว็บ (ชุดที่ 1)** ก่อนเสมอ

## ระบบรหัสผ่าน 2 ชั้น

| | ดู ค่าน้ำค่าไฟ | แก้ ค่าน้ำค่าไฟ | ดู สั่งซื้อ | แก้ สั่งซื้อ |
|--|:--:|:--:|:--:|:--:|
| **ใส่รหัสชุดที่ 1** (เข้าเว็บ) | ✓ | ✓ | ✓ | — |
| **+ ใส่รหัสชุดที่ 2** (ปุ่ม “ปลดล็อกเพื่อแก้ไข” ในหน้าสั่งซื้อ) | ✓ | ✓ | ✓ | ✓ |

- เข้าเว็บ = ใส่รหัสชุดที่ 1 ครั้งเดียว → ดู/แก้ค่าน้ำค่าไฟ และดูสั่งซื้อได้ทันที
- จะแก้/อัปเดตหน้าสั่งซื้อ → กดปุ่ม **ปลดล็อกเพื่อแก้ไข** แล้วใส่รหัสชุดที่ 2
- ปุ่ม **ล็อกหน้าจอ** (ขวาบน) = ออกจากระบบ ต้องใส่รหัสใหม่

## การใช้งาน

| ปุ่ม | ทำอะไร |
|------|--------|
| สลับ **ค่าน้ำค่าไฟ / สั่งซื้อ** | เปลี่ยนโมดูล (กลางบน) |
| ☀️ / 🌙 | สลับโหมดสว่าง/มืด |
| **ปลดล็อกเพื่อแก้ไข** (หน้าสั่งซื้อ) | ใส่รหัสชุดที่ 2 เพื่อเปิดสิทธิ์แก้สั่งซื้อ |
| **พิมพ์ช่วงที่เลือก / พิมพ์รอบนี้** | ออกใบสรุป A4 |

แก้ข้อมูลแบบ inline: คลิกที่ตัวเลขในตารางเพื่อแก้ได้ทันที (บันทึกลง Sheet อัตโนมัติ)

## ความปลอดภัย (ควรรู้)

- ข้อมูลทั้งหมด **โหลดไม่ได้ถ้าไม่มีรหัสชุดที่ 1** (ตรวจฝั่ง Apps Script) และ **แก้สั่งซื้อไม่ได้ถ้าไม่มีรหัสชุดที่ 2**
- รหัสเก็บใน Apps Script เท่านั้น ไม่ได้อยู่ใน GitHub
- เหมาะกับงานภายในทั่วไป ไม่ใช่ระบบความปลอดภัยระดับสูง
- Google Sheet ตัวจริงยังเป็นส่วนตัว (เว็บคุยผ่าน Apps Script เท่านั้น)
