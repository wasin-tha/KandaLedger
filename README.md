# KandaLedger — ค่าน้ำค่าไฟ · สั่งซื้อ · ห้องพัก แคมป์คนงาน

เว็บเดียว 3 โมดูล + ฐานข้อมูล Google Sheet + โฮสต์ฟรีบน GitHub Pages

- **โมดูลค่าน้ำค่าไฟ** — แยกตามคนงาน, มิเตอร์ไฟ/น้ำ → คำนวณหน่วย × เรต, ค่าเช่า, ค่าปรับ/อื่น ๆ, ติ๊กจ่ายแล้ว, เลือกช่วงเดือน, ย่อ/ขยายรายปี, พิมพ์ A4
- **โมดูลสั่งซื้อ** — แบ่งเป็นรอบ → หลายใบเสร็จ → หลายรายการ, แนบรูปใบเสร็จ, แคตตาล็อกสินค้า + autocomplete, ปุ่มพิมพ์ต่อรอบ
- **โมดูลห้องพัก** — จดการเข้าพักรายวัน 12 ห้อง (ชั่วคราว/ค้างคืน) แบบปฏิทินทั้งเดือน, แตะ = เซฟทันที, ทำงาน offline ได้, มีหน้า **เทียบ** หาจุดที่จดไม่ตรงกัน
- **สิทธิ์ตามบทบาท** — เจ้าของ / ปลั๊ก / แม่บ้าน คนละ PIN เห็นไม่เท่ากัน (ตรวจฝั่ง server)

🌐 **ใช้งานจริงแล้ว:** https://wasin-tha.github.io/KandaLedger/

```
KandaLedger/
├─ index.html        หน้าเว็บ (โครง + จุด mount)
├─ styles.css        ดีไซน์ระบบ (Noto Sans Thai, light/dark, print)
├─ app.js            ตรรกะทั้งหมด (state, render, API client, การพิมพ์)
├─ config.js         << วาง URL ของ Apps Script ตรงนี้ (public ได้ — ความปลอดภัยอยู่ที่ PIN)
├─ apps-script/      โค้ดฝั่ง Google (backend) — gitignore ไว้ ไม่ขึ้น public
│  └─ Code.gs
└─ README.md
```

> โครงสร้าง: front-end เป็น static vanilla JS ล้วน (ไม่มี build step) · backend เป็น Google Apps Script ผูกกับ Sheet · รูปใบเสร็จ/สินค้าเก็บใน Google Drive

## 🔑 ระบบสิทธิ์ตามบทบาท (1 คน 1 PIN)

แทนระบบรหัส 2 ชั้นแบบเดิม — ตอนนี้แต่ละคนมี PIN ของตัวเอง server ดูจาก PIN ว่าเป็นใครแล้วเปิดสิทธิ์ให้ตามตาราง:

| บทบาท | ค่าน้ำค่าไฟ | สั่งซื้อ | ห้องพัก |
|------|:--:|:--:|:--:|
| 👑 **เจ้าของ** (March) | ดู + แก้ | เต็ม (แก้/ลบ/ตั้งเรต) | จด + ดูทุกฝ่าย + เทียบ |
| 🔧 **ปลั๊ก** | ดู + แก้ | ดู + กด “จัดการแล้ว” (แก้ไม่ได้) | จดของปลั๊ก + ดูแม่บ้าน + เทียบ |
| 🧹 **แม่บ้าน** | — | — | จดของแม่บ้านอย่างเดียว |

- PIN ทั้ง 3 เก็บใน **Script Properties** คีย์ `PIN_OWNER` / `PIN_PLUG` / `PIN_MAID` — **ไม่อยู่ในซอร์สโค้ด** (กันรหัสรั่วเมื่อ repo เป็น public)
- เปลี่ยนรหัส = แก้ค่าใน Properties ได้เลย ไม่ต้อง push ใหม่
- ใส่รหัสผิดจะหน่วงเวลา ~1.2 วิ (กันเดารหัส) · URL เว็บแอปเป็น public เสมอ → **ตั้งรหัสยาว 8–10 ตัวผสมตัวอักษร + เลข**
- ปุ่ม **ล็อกหน้าจอ** (ขวาบน) = ออกจากระบบ ล้าง PIN ที่จำไว้ ต้องใส่ใหม่

> ⚠️ ต้องตั้ง `PIN_OWNER` / `PIN_PLUG` / `PIN_MAID` ใน Script Properties ก่อน ทุกคนถึงจะ login ด้วยรหัสตัวเองได้ (โดยเฉพาะแม่บ้าน ไม่มีรหัสสำรอง)

## คู่มือ deploy (เผื่อต้องตั้งใหม่ทั้งหมด)

### ขั้นที่ 1 — ตั้งค่า Google Sheet + Apps Script (ทำครั้งเดียว)

1. สร้าง Google Sheet เปล่าใน Drive ของคุณ → เมนู **Extensions ▸ Apps Script**
2. ลบโค้ดตัวอย่างทั้งหมด แล้วก๊อปปี้เนื้อหา `apps-script/Code.gs` ไปวางแทน → **Save** (💾)
3. ตั้งรหัสที่ **Project Settings (⚙️) ▸ Script Properties**:
   - `PIN_OWNER` = รหัสเจ้าของ (เห็นทุกอย่าง)
   - `PIN_PLUG` = รหัสปลั๊ก
   - `PIN_MAID` = รหัสแม่บ้าน
4. เลือกฟังก์ชัน **`setup`** จาก dropdown ด้านบน → กด **Run**
   - ครั้งแรกจะขอสิทธิ์ → เลือกบัญชีคุณ → **Advanced** → **Go to … (unsafe)** → **Allow**
   - เสร็จแล้วจะเห็นแท็บต่าง ๆ ใน Sheet (`Workers`, `Bills`, `Rounds`, `Items`, `Images`, `Products` + แท็บห้องพัก `Rentals`, `Rates`, `Notes` สร้างอัตโนมัติเมื่อเริ่มใช้)

> ⚠️ `setup` เป็นฟังก์ชัน **ล้าง + ใส่ข้อมูลตั้งต้นใหม่ทุกแท็บ** — รันแค่ตอนตั้งระบบครั้งแรกเท่านั้น อย่ารันทับข้อมูลจริง

### ขั้นที่ 2 — Deploy เป็น Web App

1. กด **Deploy ▸ New deployment** → เฟือง ⚙️ ข้าง “Select type” → **Web app**
2. ตั้งค่า **Execute as:** `Me` · **Who has access:** `Anyone` ← สำคัญ (ให้เว็บเรียกได้)
3. กด **Deploy** → คัดลอก **Web app URL** (ลงท้าย `/exec`)

> แก้ `Code.gs` แล้วอยากให้มีผล: **Deploy ▸ Manage deployments ▸** (ดินสอ) **▸ Version: New version ▸ Deploy** (URL เดิมไม่เปลี่ยน)
> ถ้าใช้ clasp: `cd apps-script && clasp push -f && clasp create-deployment -i <DEPLOYMENT_ID> -d "ข้อความ"`

### ขั้นที่ 3 — ใส่ URL ลงเว็บ

เปิด `config.js` แล้ววาง URL:

```js
window.KANDA_CONFIG = {
  WEB_APP_URL: "https://script.google.com/macros/s/AKfyc..../exec",
  ORG_NAME: "แคมป์คนงาน"
};
```

เปิด `index.html` ในเบราว์เซอร์เพื่อทดสอบ — แบนเนอร์ “โหมดตัวอย่าง” ต้องหายไป แล้วหน้า login ต้องขึ้น

### ขั้นที่ 4 — ขึ้น GitHub Pages (ฟรี)

```bash
cd KandaLedger
git init
git add .                # apps-script/ + CLAUDE.md ถูก gitignore ไว้แล้ว ไม่ต้องกลัวหลุด
git commit -m "KandaLedger web app"
git branch -M main
git remote add origin https://github.com/<user>/KandaLedger.git
git push -u origin main
```

แล้วไปที่ repo → **Settings ▸ Pages** → **Source:** Deploy from a branch → **Branch:** `main` / `/ (root)` → Save
รอ ~1 นาที จะได้ลิงก์ `https://<user>.github.io/KandaLedger/` — เปิดมาจะถามรหัสก่อนเสมอ

## การใช้งาน

| ปุ่ม | ทำอะไร |
|------|--------|
| สลับ **ค่าน้ำค่าไฟ / สั่งซื้อ / ห้องพัก** (กลางบน) | เปลี่ยนโมดูล (เห็นเฉพาะที่บทบาทมีสิทธิ์) |
| ☀️ / 🌙 | สลับโหมดสว่าง/มืด |
| **พิมพ์ช่วงที่เลือก / พิมพ์รอบนี้** | ออกใบสรุป A4 |
| **ล็อกหน้าจอ** (ขวาบน) | ออกจากระบบ |

- แก้ข้อมูลแบบ inline: คลิกที่ตัวเลขในตารางเพื่อแก้ได้ทันที (บันทึกลง Sheet อัตโนมัติ) · ช่องตัวเลขพิมพ์นิพจน์ `+`/`−` ได้ (เช่น `91+161`)
- หน้าห้องพัก: แตะวันในปฏิทิน → ช่องจดโผล่ใต้ปฏิทิน, แตะปรับเลข = เซฟทันที (มีจุดบอกสถานะซิงค์, จดตอนเน็ตหลุดได้แล้วค่อยส่งทีหลัง)
- เว็บ auto-sync ทุก ~10 วิ — เปิดหลายเครื่องพร้อมกันเห็นข้อมูลตรงกัน

## ความปลอดภัย (ควรรู้)

- ข้อมูลทั้งหมด **โหลดไม่ได้ถ้าไม่มี PIN** และ server กรองให้เห็นเฉพาะส่วนที่บทบาทนั้นมีสิทธิ์ (ตรวจฝั่ง Apps Script ไม่ใช่แค่ซ่อนปุ่ม)
- ทุก request เป็น POST → PIN ไม่โผล่ใน URL/log
- รหัสเก็บใน Script Properties เท่านั้น ไม่ได้อยู่ใน GitHub · `apps-script/` (มี seed ข้อมูลจริง) และ `CLAUDE.md` (มีโน้ตภายใน) ถูก gitignore ไม่ขึ้น public
- Google Sheet ตัวจริงยังเป็นส่วนตัว (เว็บคุยผ่าน Apps Script เท่านั้น)
- เหมาะกับงานภายในทั่วไป ไม่ใช่ระบบความปลอดภัยระดับสูง
